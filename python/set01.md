# Execution Strategy

For each problem:

1. Solve naively first.
2. Analyze time and space complexity.
3. Refactor for readability.
4. Add:

   * type hints
   * logging
   * validation
   * error handling
   * tests
5. Benchmark performance where relevant.
6. Package the solution cleanly.
7. Push to GitHub with README and architecture notes.

---

# Recommended Standards

For backend/systems-oriented solutions:

* Use `typing`
* Use `dataclasses` where appropriate
* Prefer `pathlib` over raw paths
* Use `pydantic` for validation
* Use structured logging
* Add unit tests with `pytest`
* Use virtual environments
* Follow modular project structure
* Add linting (`ruff`) and formatting (`black`)
